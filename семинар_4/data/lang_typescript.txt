TypeScript — язык программирования

TypeScript — язык программирования, представленный компанией Microsoft в 2012 году. Его главным архитектором является Андерс Хейлсберг — тот же инженер, который ранее спроектировал язык C# и среду Turbo Pascal/Delphi. TypeScript создавался как ответ на проблемы крупных кодовых баз на JavaScript, где отсутствие статической типизации затрудняло поддержку и рефакторинг больших проектов.

Ключевая идея языка в том, что TypeScript является строгим надмножеством (супермножеством) JavaScript: любой корректный код на JavaScript одновременно является корректным кодом на TypeScript. К возможностям JavaScript TypeScript добавляет необязательную статическую типизацию, аннотации типов, интерфейсы, перечисления (enum), обобщённые типы (generics) и модификаторы доступа. Это позволяет обнаруживать целые классы ошибок ещё на этапе написания кода, а не во время выполнения.

Сам по себе браузер не умеет исполнять TypeScript напрямую. Поэтому код на TypeScript проходит этап компиляции (точнее, транспиляции): компилятор tsc переводит его в обычный JavaScript, который уже запускается в браузере или в среде Node.js. При этом аннотации типов полностью стираются при компиляции и не влияют на поведение программы во время выполнения — это так называемая «стираемая» типизация. Система типов TypeScript является структурной: совместимость двух типов определяется их структурой (набором полей и методов), а не явно объявленным наследованием.

Поскольку итоговым результатом всегда оказывается JavaScript, TypeScript наследует его модель исполнения: однопоточность с циклом событий, автоматическое управление памятью со сборщиком мусора и прототипную объектную модель под капотом. Файлы с исходным кодом имеют расширение .ts (или .tsx для кода с JSX-разметкой).

TypeScript получил очень широкое распространение в индустрии. Многие популярные фреймворки, например Angular, по умолчанию написаны и используются именно на TypeScript, а React и Vue имеют первоклассную поддержку типов. Язык особенно ценят в больших командах: статические типы служат документацией, улучшают автодополнение в редакторах кода и снижают риск ошибок при изменении общего кода. Конфигурация компилятора задаётся в файле tsconfig.json, где можно включать строгие проверки, например запрет неявного типа any.

Пример типизированной функции на TypeScript:

function greet(name: string): string {
  return `Hello, ${name}!`;
}

Таким образом, TypeScript не заменяет JavaScript, а расширяет его, добавляя слой статической безопасности типов поверх уже существующей экосистемы и сохраняя полную совместимость с ней.

История TypeScript — это история постепенного завоевания доверия сообщества. После выхода в 2012 году язык поначалу встречали скептически, но по мере роста сложности веб-приложений преимущества статической типизации становились всё очевиднее. Крупные проекты с открытым исходным кодом и большие компании массово переходили на TypeScript ради удобства поддержки. Система типов языка развивалась и стала одной из самых выразительных среди мейнстримных языков: появились условные типы, сопоставление типов (mapped types), шаблонные литеральные типы, что позволяет описывать очень точные ограничения на данные.

Важно понимать, что TypeScript не даёт никаких гарантий во время выполнения: поскольку типы стираются при компиляции, ошибочные данные, пришедшие извне (например, из сетевого запроса), всё равно могут нарушить ожидания, если их не проверить вручную. Поэтому типы TypeScript защищают в первую очередь от ошибок самого программиста на этапе разработки, а не от некорректных данных в рантайме. Тем не менее сочетание полной совместимости с JavaScript, мощной системы типов и отличной поддержки в редакторах сделало TypeScript стандартом для большинства серьёзных фронтенд- и многих бэкенд-проектов.

Стоит отметить и инструментальную сторону: глубокая интеграция TypeScript с редакторами кода (прежде всего с Visual Studio Code, который сам написан на TypeScript) даёт мгновенное автодополнение, навигацию по коду и подсветку ошибок прямо во время набора. Для описания типов внешних библиотек, написанных на чистом JavaScript, существует репозиторий деклараций типов DefinitelyTyped, благодаря которому даже нетипизированные пакеты можно использовать типобезопасно. Эта экосистема инструментов и определений типов стала одной из причин повсеместного принятия языка.

Java — язык программирования

Java — строго типизированный объектно-ориентированный язык программирования общего назначения. Он был разработан компанией Sun Microsystems под руководством Джеймса Гослинга и официально выпущен в 1995 году. Изначально проект назывался Oak и предназначался для программирования бытовой электроники, но в итоге язык нашёл своё главное применение в корпоративной разработке и в вебе. В 2010 году вместе с компанией Sun язык перешёл под управление корпорации Oracle.

Главная идея Java выражена девизом «Write once, run anywhere» («напиши один раз, запускай где угодно»). Достигается это за счёт промежуточного представления: исходный код компилируется не в машинный код конкретного процессора, а в платформо-независимый байт-код. Этот байт-код исполняется виртуальной машиной Java (JVM, Java Virtual Machine), которая существует для разных операционных систем. Современные JVM применяют JIT-компиляцию, переводя горячие участки байт-кода в нативный машинный код во время исполнения, что обеспечивает высокую производительность.

Управление памятью в Java полностью автоматическое: за освобождение неиспользуемых объектов отвечает сборщик мусора, а прямого доступа к указателям у программиста нет. Это устраняет целый класс ошибок, связанных с ручным управлением памятью. Система типов статическая и строгая. Язык изначально проектировался как объектно-ориентированный: почти всё является объектом, поддерживаются классы, интерфейсы, одиночное наследование классов и множественная реализация интерфейсов. Примитивные типы (int, double, boolean и другие) вынесены отдельно ради эффективности.

Со временем язык заметно эволюционировал. В версии Java 5 появились обобщённые типы (generics) и аннотации, в Java 8 — лямбда-выражения и потоковый API (Stream API), привнёсшие в язык функциональный стиль. С недавних пор Oracle перешла на быстрый цикл релизов: новая версия выходит каждые полгода, а отдельные версии получают статус долгосрочной поддержки (LTS).

Файлы с исходным кодом имеют расширение .java, а скомпилированный байт-код — расширение .class. Экосистема Java очень зрелая: системы сборки Maven и Gradle, фреймворк Spring для серверных приложений, огромное число библиотек. Язык доминирует в корпоративной разработке, банковских системах и крупных бэкенд-сервисах, а также является основой платформы Android (хотя там его постепенно теснит Kotlin).

Пример программы «Hello, world!» на Java традиционно многословен и требует объявления класса и метода main:

public class Main {
    public static void main(String[] args) {
        System.out.println("Hello, world!");
    }
}

Сочетание переносимости через JVM, автоматического управления памятью и огромной экосистемы обеспечило Java место одного из самых распространённых языков корпоративной разработки на протяжении десятилетий.

Экосистема вокруг Java — одна из самых больших и зрелых в индустрии. На виртуальной машине Java исполняется не только сам язык Java, но и целое семейство других языков: Kotlin, Scala, Groovy, Clojure — все они компилируются в тот же байт-код и пользуются общими библиотеками. Это превращает JVM в полноценную платформу, а не просто среду исполнения одного языка. Серверный фреймворк Spring, системы сборки Maven и Gradle, серверы приложений и огромное число библиотек образуют надёжную основу для корпоративных систем.

У языка есть и устойчивая критика. Java традиционно считается многословной: даже простая программа требует объявления класса и метода main, а до появления лямбда-выражений в Java 8 многие шаблонные задачи решались громоздко. Запуск виртуальной машины и прогрев JIT-компилятора создают накладные расходы при старте, что неудобно для короткоживущих процессов и бессерверных функций; частично эту проблему решает опережающая (AOT) компиляция в нативный код через проект GraalVM. Несмотря на это, сочетание переносимости, стабильности и колоссальной экосистемы удерживает Java в числе ведущих языков корпоративной разработки уже несколько десятилетий.
